Methods

※各メソッドを利用の際には必須となるパラメータと任意のものとがあります。

blogEntry.comments

指定したblog記事のコメントの一覧が返されます。
パラメータ 説明
blogEntryId (必須) 各blogの記事に割り当てられたID 例:/blog/entry/79
返り値：指定した記事のコメント(Comment) 例：http://codeforces.com/api/blogEntry.comments?blogEntryId=79

blogEntry.view

指定したblog記事の本文が返されます。
パラメータ 説明
blogEntryId (必須) 各blogの記事に割り当てられたID 例:/blog/entry/79
返り値：指定した記事の本文(BlogEntry) 例：http://codeforces.com/api/blogEntry.view?blogEntryId=79

contest.hacks

指定したコンテストのhackの結果が返されます。完全な情報はコンテスト終了から一定時間後になります。コンテスト中は自身のhackのみ閲覧が可能です。
パラメータ 説明
contestId (必須) 各コンテストに割り当てられたID ラウンド数とは異なります。 例:/contest/566
返り値：指定したコンテストのhack。(A,B,C...など全問題が対象になるので注意) 例：http://codeforces.com/api/contest.hacks?contestId=566

contest.list

指定したコンテストの情報が得られます。
パラメータ 説明
gym bool型 trueを指定でgymの情報が得られる。それ以外の場合は通常のコンテストの情報が得られる。
返り値：指定したコンテストの情報(Contest) 例：http://codeforces.com/api/contest.list?gym=true  gymのパラメータを省略した場合はfalseとなる。

contest.ratingChanges

指定したコンテスト終了後のレーティングの変動が得られます。
パラメータ 説明
contestId (必須) 各コンテストに割り当てられたID ラウンド数とは異なります。 例:/contest/566/status
返り値：指定したコンテストの終了後のレーティングの変動が得られます。 (RatingChange) 例：http://codeforces.com/api/contest.ratingChanges?contestId=566

contest.standings

指定したコンテスト終了後の参加者の順位が得られます。
パラメータ 説明
contestId (必須) 各コンテストに割り当てられたID、ラウンド数とは異なります。 例:/contest/566
from 1位からの場合にはfrom=1で順位の始まりを指定します。
count 開始位置から追加で何位まで見るかを指定します。
handles セミコロン区切りで任意の参加者(10000人まで)を指定します。from〜countとは同時に指定出来ない？
room 該当のroomのデータが得られます。
showUnofficial trueで指定した場合にはvirtual参加者、div違い、レーティング対象外者などを含みます。
返り値："contest", "problems" , "rows"の3つの要素があります。contestはcontest。problemsはproblems。rowsはrows。 例：http://codeforces.com/api/contest.standings?contestId=566&from=1&count=5&showUnofficial=true

contest.status

指定したコンテストの提出結果が得られます。
パラメータ 説明
contestId (必須) 各コンテストに割り当てられたID、ラウンド数とは異なります。 例:/contest/566/status
handle codeforcesユーザーのハンドルネームを指定します。
from 1が最新のもので始まりを指定します。
count 何件分の提出かを指定します。
返り値：Submissionが得られます。降順でソートされています。 例：http://codeforces.com/api/contest.status?contestId=566&from=1&count=10

problemset.problems

出題済の問題が返されます。tagでフィルタリングが出来ます。
パラメータ 説明
tags セミコロン区切りで複数タグ指定が出来ます。
返り値： Problem と ProblemStatistics が得られます。 例：http://codeforces.com/api/problemset.problems?tags=dp;math

problemset.recentStatus

最近の提出が得られます。from指定が使えないようです。あくまでも最近(recent)の提出を見るもののようです。
パラメータ 説明
count (必須) 件数を指定します。上限は1000件です。
返り値： Submission が得られます。 最近の提出IDから降順でソートされています。 例：http://codeforces.com/api/problemset.recentStatus?count=10

recentActions

codeforcesのblogの新着記事、新着コメントが得られます。
パラメータ 説明
maxCount (必須) 件数を指定します。上限は100件です。
返り値： RecentAction が得られます。 from、新着記事のみ、新着コメントのみなどの指定は出来ないようです。 例：http://codeforces.com/api/recentActions?maxCount=30

user.blogEntries

該当のユーザーのcodeforcesのblogの情報が得られます。
パラメータ 説明
handle (必須) ハンドルネームを指定します。
返り値： BlogEntry が得られます。 情報は省略版？？。 例：http://codeforces.com/api/user.blogEntries?handle=Fefer_Ivan

user.friends

該当のユーザーのフレンド情報が得られます。認証が必要です。
パラメータ 説明
onlyOnline bool型 trueを指定でオンラインのユーザーのみにフィルタリング。
返り値：ユーザーのハンドルネーム が得られます。 api keyが必要になります。 例：http://codeforces.com/api/user.friends?onlyOnline=true

user.info

該当のユーザーの情報が得られます。
パラメータ 説明
handles (必須) ハンドルネームでユーザーを指定します。 セミコロン区切りで上限は10000件です。
返り値：該当ユーザーの User が得られます。 例：http://codeforces.com/api/user.info?handles=DmitriyH;Fefer_Ivan

user.ratedList

ユーザーのレーティングが得られます。
パラメータ 説明
activeOnly bool型 trueを指定で先月までに参加履歴のあるユーザーのみにフィルタリングします。
返り値：User が得られます。レーティング値で降順(順位では昇順)でソートされています。 例：http://codeforces.com/api/user.ratedList?activeOnly=true

user.rating

該当ユーザーのレーティングの履歴が得られます。
パラメータ 説明
handle (必須) ハンドルネームでユーザーを指定します。
返り値： RatingChange が得られます。古->新でソートされているようです。 例：http://codeforces.com/api/user.rating?handle=Fefer_Ivan

user.status

該当ユーザーの提出の履歴が得られます。
パラメータ 説明
handle (必須) ハンドルネームでユーザーを指定します。
from 1が最新のもので始まりを指定します。
count 件数を指定します。
返り値： 該当ユーザーの Submission が得られます。提出の新しいものから降順にソートされています。 例：http://codeforces.com/api/user.status?handle=Fefer_Ivan&from=1&count=10
